📖 प्रयोग निर्देशन — NEA Taulihawa DCS

१. सन्देश पठाउने (Entry): सबस्टेशन → अपरेटर → फिडर → अवस्था (Trip / Shutdown) → कारण र समय छानेर Dispatch थिच्नुहोस्। एउटै सन्देश सबै रिसिभरहरूमा (WhatsApp/SMS) जान्छ। फिडर चार्ज भएपछि Charged थिच्नुहोस् — restore सन्देश स्वतः जान्छ।

२. सिस्टम गन (System Gone): अवस्थामा ⛔ System Gone छान्नुहोस्, प्रभावित सबस्टेशनहरूमा मात्र ✔ लगाउनुहोस् (नबिग्रेकोमा ✔ हटाउनुहोस्) र Dispatch गर्नुहोस्। पुनर्स्थापना गर्न प्रत्येक सबस्टेशनको रातो ब्यानरमा System Restored थिच्नुहोस्।

३. इनकमर फेल: 🛑 11 KV Incomer Failed ले त्यस सबस्टेशनका सबै फिडर Incomer Failed देखाउँछ; ⚡ 33 KV Incomer Failed ले सबै फिडर Trip गराउँछ। दुबैको Restore ब्यानरबाट गर्नुहोस् — restore सन्देश स्वतः जान्छ।

४. डाउनटाइम रिपोर्ट (Log पेज): हरेक Trip / Shutdown / System Gone / Incomer Failed को सुरु–अन्त्य समय र अवधि स्वतः रेकर्ड हुन्छ र सबै अपरेटर डिभाइसहरूबीच सिंक हुन्छ। दैनिक रिपोर्ट तोकिएको समयमा (पूर्वनिर्धारित १०:००) स्वतः Excel + text सहित पठाइन्छ — टगलबाट बन्द/खोल्न र समय बदल्न सकिन्छ।

५क. वास्तविक समय: System Gone / Incomer Failed / Trip All प्रायः केही मिनेटपछि थाहा हुन्छ — dispatch गर्दा "वास्तविक समय" फिल्डमा र restore गर्दा देखिने प्रश्नमा घटना भएकै समय लेख्नुहोस्; सन्देश, लग र रिपोर्ट सबैमा त्यही सही समय जान्छ।

५ख. समूह सच्याइ: System Gone जस्ता घटनाले धेरै फिडर एकसाथ समेट्छ (तालिकामा ×N चिन्ह)। एउटा फिडरको समय सच्याउँदा सोधिन्छ — OK थिचे सबै सम्बन्धित फिडरमा एउटै समय लागू हुन्छ; Cancel थिचे त्यो फिडरमा मात्र। मिति पनि बदल्न मिल्ने भएकाले जति लामो अवधि पनि राख्न सकिन्छ। सबै परिवर्तन स्थायी हुन्छन् र सबै डिभाइसमा सिंक हुन्छन्।

५. रिपोर्ट सच्याउने: तालिकाका सुरु/अन्त्य समय सिधै edit गर्न सकिन्छ, गल्ती प्रविष्टि 🗑 बाट हटाउन सकिन्छ, छुटेको घटना ＋ Add Missed Entry बाट थप्न सकिन्छ। ⚠ चिन्ह भएका पङ्क्तिको अन्त्य समय अनुमानित हो — पठाउनुअघि जाँच्नुहोस्। Send Report Now थिच्दा पहिले समीक्षा (Review) देखिन्छ — जाँचेर मात्र Confirm & Dispatch गर्नुहोस्।

५ग. दुई गणना: रिपोर्टमा समूह घटनाहरू दुई तरिकाले देखिन्छन् — मुख्य घटना / फिडर-गुणा। जस्तै "System Gone: 1 (3h 57m) / 10 (39h 30m)" = System Gone १ पटक भयो (३ घ. ५७ मि.), जसले १० फिडर-घटना (कुल ३९ घ. ३० मि.) समेट्यो। एकल Trip/Shutdown मा दुबै बराबर हुने भएकाले एउटै अङ्क देखिन्छ।

५घ. समूह फिडर छनोट: Settings → Group Outage Feeders मा गएर System Gone / 11 KV / 33 KV / Trip All ले कुन-कुन फिडर असर गर्छ भनेर सबस्टेशन-अनुसार छान्नुहोस् (पूर्वनिर्धारित = सबै)। त्यसपछि ती घटनाहरूले छानिएका फिडरमा मात्र status, message र downtime लागू गर्छन्।

६. साप्ताहिक / मासिक: अवधि छान्नुहोस् (Weekly = ७ दिन, Monthly = पूरा महिना) — Excel मा दिनैपिच्छेको ब्रेकडाउन पाना पनि आउँछ। पुरानो इतिहास ⟲ Import Past History बाट message log बाट आउँछ।

७. सक्रिय अवरोध सूचना: सबस्टेशन छान्दा वा हरेक १० मिनेटमा, त्यस सबस्टेशनका सक्रिय Trip/Shutdown/System Gone को नेपाली सूचना देखिन्छ — कुनै फिडर restore गर्न नबिर्सनुहोस्।

सुझाव: सधैँ खुला रहने अफिस डिभाइसमा auto-send ON राख्नुहोस्; सबै डिभाइसमा एउटै रिपोर्ट समय राख्नुहोस्।
